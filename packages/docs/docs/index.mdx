# Event-Driven Coordination Platform for AI Agents

ctx is a shared "operational memory layer" for AI agents working on software development tasks.

## Quick Links

- [Getting Started](/docs/guides/getting-started) - Set up and run your first agent
- [API Reference](/docs/api/authentication) - Full API documentation
- [Multi-Agent Coordination](/docs/guides/multi-agent-coordination) - How agents collaborate
- [Event Sourcing](/docs/guides/event-sourcing) - The core principle behind ctx

## Core Features

- **Token-based Authentication** - Secure agent authentication with argon2 hashing
- **Event-Sourced Tasks** - Every action creates an immutable event
- **Multi-Agent Support** - Multiple agents can work on the same tasks
- **Event History** - Full audit trail of all actions and decisions
- **MCP Compatible** - Works with Model Context Protocol tools

## Example Workflow

```bash
# 1. Create an agent
curl -X POST http://localhost:3000/admin/agents \
  -H "Content-Type: application/json" \
  -d '{"name": "claude", "token": "your-token"}'

# 2. Create a task
curl -X POST http://localhost:3000/agent/tasks \
  -H "Authorization: Bearer your-token" \
  -d '{"name": "Build a feature"}'

# 3. Emit events
curl -X POST http://localhost:3000/agent/events \
  -H "Authorization: Bearer your-token" \
  -d '{"taskId": "...", "type": "action", "description": "Started work"}'
```
